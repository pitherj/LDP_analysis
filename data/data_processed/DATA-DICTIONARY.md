# Data Dictionary — data/data_processed/

Derived files written by `scripts/full_workflow.qmd` during analysis. These files are not manually created and should be regenerated by rendering the Quarto document rather than edited directly.

`crossref_journal_cache.rds` is the one exception: it is written on the first render and then read from cache on all subsequent renders to avoid repeated API calls. Delete it to force a fresh CrossRef query.

---

## Directory Contents

```
data_processed/
├── pairs_lookup.csv            # Non-sensitive pairing key (pub_id, pair_id, group)
├── rater_scores_wide.csv       # FAIR scores in wide format (one row per publication)
├── pairs_data.csv              # One row per matched pair with paired differences
└── crossref_journal_cache.rds  # CrossRef API cache (journal metadata; auto-generated)
```

---

## File: pairs_lookup.csv

A privacy-safe subset of the private pairing key, containing only the three columns needed for the primary analysis. Sensitive author and institution information from the source key file is excluded.

**Rows:** 38 (one per publication common to all four raters)  
**Generated by:** `scripts/full_workflow.qmd`, chunk `load-key` (hidden; `echo: false`)  
**Source file:** `data/data_raw/data_private/rater_key_final_v2.csv`  
**Git-ignored:** no — contains no identifiable information  
**Regenerate:** render `full_workflow.qmd`; the hidden chunk overwrites this file on each run

| Column | Type | Units | Description | Nullable |
|---|---|---|---|---|
| `pub_id` | character | — | Unique publication identifier (6-character alphanumeric); foreign key linking to rater files | no |
| `pair_id` | character | — | Matched-pair identifier (e.g., `PAIR001`); two records share each `pair_id`, one LDP and one Comparator; IDs are not necessarily contiguous (2 pairs were dropped post-hoc) | no |
| `group` | character | — | Group membership: `LDP` (trained student) or `Comparator` (matched control) | no |

**Integrity anchor:** 38 rows; 19 distinct `pair_id` values; exactly one `LDP` and one `Comparator` record per `pair_id`.

## File: rater_scores_wide.csv

FAIR compliance scores for the 38 publications common to all four raters, in wide format with one column per rater. This file feeds the inter-rater reliability analyses.

**Rows:** 38 (one per publication)  
**Columns:** 5 (`pub_id` + four rater score columns)  
**Generated by:** `scripts/full_workflow.qmd`, chunk `reformat-data`  
**Source files:** all four files in `data/data_raw/`  
**Git-ignored:** no  
**Regenerate:** render `full_workflow.qmd`; the chunk overwrites this file on each run

| Column | Type | Units | Description | Nullable |
|---|---|---|---|---|
| `pub_id` | character | — | Unique publication identifier (6-character alphanumeric) | no |
| `JP` | numeric | points | Rater JP's composite FAIR compliance score (range 0–4) | no |
| `DH` | numeric | points | Rater DH's composite FAIR compliance score (range 0–4) | no |
| `JR` | numeric | points | Rater JR's composite FAIR compliance score (range 0–4); sourced from `total` column in original file | no |
| `SE` | numeric | points | Rater SE's composite FAIR compliance score (range 0–4) | no |

**Integrity anchor:** 38 rows; all score values in [0, 4]; no missing values expected.

**Note on score granularity:** Non-integer values are expected. The Findable component can take the value 0.5 (unstructured data availability statement), and each of the five Reusable sub-items contributes 0.2, so valid total scores include values such as 0.2, 0.4, 0.5, 0.6, 0.8, 1.0, 1.2, etc.

## File: pairs_data.csv

One row per matched pair containing group mean FAIR scores and the within-pair difference. This is the primary input to the power analysis comparison (variability assumptions section).

**Rows:** 19 (one per matched pair)  
**Generated by:** `scripts/full_workflow.qmd`, chunk `prep-data`  
**Git-ignored:** no  
**Regenerate:** render `full_workflow.qmd`

| Column | Type | Units | Description | Nullable |
|---|---|---|---|---|
| `pair_id` | character | — | Matched-pair identifier (e.g., `PAIR001`); foreign key to `pairs_lookup.csv` | no |
| `LDP` | numeric | points | Mean FAIR compliance score for the LDP publication in this pair (mean across 4 raters; range 0–4) | no |
| `Comparator` | numeric | points | Mean FAIR compliance score for the Comparator publication in this pair (mean across 4 raters; range 0–4) | no |
| `D` | numeric | points | Within-pair difference: LDP − Comparator | no |

**Integrity anchor:** 19 rows; `D` = `LDP` − `Comparator` exactly; all values in [−4, 4].

## File: crossref_journal_cache.rds

A serialised R list caching CrossRef REST API responses for all 38 publication DOIs. Each list element is named by `pub_id` and contains the journal name, ISSN(s), and publisher retrieved from CrossRef.

**Format:** R binary (`.rds`); read with `readRDS()`  
**Generated by:** `scripts/full_workflow.qmd`, chunk `jp-crossref-query` (on first render only)  
**Git-ignored:** no — included so re-renders do not require internet access or repeated API calls  
**Regenerate:** delete this file and re-render `full_workflow.qmd`; the chunk queries the CrossRef API (polite rate-limiting: 0.5 s between requests) and writes a fresh cache

**Structure:** named list of length 38; each element has fields:

| Field | Type | Description |
|---|---|---|
| `journal` | character | Journal name as returned by CrossRef (`container-title`) |
| `issns` | character vector | All ISSNs associated with the work (print and electronic) |
| `publisher` | character | Publisher name as returned by CrossRef |

Any field is `NA` (or empty vector for `issns`) if CrossRef did not return a value for that DOI.
